# Guía de Deploy en Hostinger

## Problema Común: "Your requirements could not be resolved"

Este error generalmente ocurre porque:
1. Hostinger puede tener una versión de PHP diferente a la requerida (PHP 8.2+)
2. El `composer.lock` puede estar desactualizado
3. Las dependencias de desarrollo pueden causar conflictos

## Soluciones:

### 1. Verificar versión de PHP en Hostinger
- Ve al panel de Hostinger → PHP Configuration
- Asegúrate de tener PHP 8.2 o superior activado
- Si no está disponible, contacta a soporte para habilitarlo

### 2. Actualizar composer.lock antes del deploy
```bash
composer update --no-dev
git add composer.lock
git commit -m "Update composer.lock for production"
git push origin main
```

### 3. Configurar Hostinger para usar --no-dev
En la configuración de deploy de Hostinger, asegúrate de que el comando sea:
```bash
composer install --no-dev --optimize-autoloader
```

### 4. Si Hostinger no tiene PHP 8.2+
Si Hostinger solo tiene PHP 8.1 o inferior, necesitarás:
- Contactar a soporte para actualizar PHP
- O ajustar el `composer.json` para ser compatible con PHP 8.1 (no recomendado)

## Pasos recomendados para deploy:

1. **Actualizar composer.lock localmente:**
   ```bash
   composer update --no-dev
   ```

2. **Compilar assets:**
   ```bash
   npm run build
   ```

3. **Commit y push:**
   ```bash
   git add .
   git commit -m "Prepare for production deployment"
   git push origin main
   ```

4. **En Hostinger, configurar:**
   - PHP Version: 8.2 o superior
   - Composer Command: `composer install --no-dev --optimize-autoloader --no-interaction`
   - Build Command: `npm run build` (si está disponible)

5. **Después del deploy, ejecutar en Hostinger (vía SSH o panel):**
   ```bash
   php artisan config:cache
   php artisan route:cache
   php artisan view:cache
   php artisan storage:link
   php artisan migrate --force
   ```
